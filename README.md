# 星恒BMS一线通协议 Saleae 解析插件
适配**星恒智能BMS一线通协议A1.3.3版本**（2023.07.18正式版），基于Saleae Logic 2开发的高电平解析插件，支持协议帧解析、公有报文全字段解析、私有报文识别、校验验证，完美匹配无锡/北京团标要求。

## 插件特性
1. 严格遵循星恒A1.3.3协议帧结构：**同步信号(T1≥10ms+T2=1ms) + 主报文 + 停止信号(N≥50ms)**
2. 支持3类核心报文解析：
   - 公有报文(0x01)：全字段解析+物理值换算+和校验
   - 单串电压报文(0x3B)：北京团标，60s发送1帧
   - 电池唯一码报文(0x3C)：北京团标，60s发送1帧(DETD接入先发数帧)
3. 适配协议关键规则：
   - 数据位先传**最低有效位**，多字节采用**Intel LSB小端模式**
   - 物理值按协议公式 `P=C×R+F` 自动换算
   - 和校验验证（所有字节之和的低8位）
4. 友好的Saleae展示：表格列直接显示核心值，详情面板查看全字段，错误帧标注具体原因

## 协议匹配版本
- 星恒智能BMS一线通协议 **A1.3.3（2023.07.18正式版）**
- 向下兼容A1.3.0/A1.3.1/A1.3.2版本
- 支持无锡团标（公有报文）、北京团标（单串电压/唯一码报文）

## 安装步骤
### 环境要求
- Saleae Logic 2 软件：≥V2.3.0（推荐最新版）
- 硬件：Saleae Logic 4/8/16 逻辑分析仪
- 采样率：建议**1Msps**（满足1ms级脉宽检测精度）

### 插件安装
1. 下载插件包，包含3个核心文件：
   - `analyzer_manifest.json` （插件配置文件）
   - `xingheng_bms_single_wire.py` （插件主程序）
   - `README.md` （使用说明）
2. 打开Saleae Logic 2 → 点击右侧「Analyzers」→ 「Add Analyzer」→ 「Custom High-Level Analyzer」
3. 选择「Load from Folder」→ 选中插件包根目录 → 插件自动加载，名称为**星恒BMS一线通协议**
4. 选中加载的插件，配置**一线通信号通道**（默认通道0，根据实际接线修改）

## 使用方法
### 硬件接线
1. 逻辑分析仪**GND** 与 星恒BMS/整车**GND** 共地（必须，否则信号干扰）
2. 逻辑分析仪**信号通道** 接 星恒BMS的**COM引脚**（一线通信号输出线）
3. 确保BMS与ECU/充电器正常连接（BMS检测到上拉电阻1s后开始发送报文）

### 采集与解析
1. 设置采样率为**1Msps**，点击「Start」开始采集
2. 插件自动解析帧结构，Saleae右侧面板显示解析结果，分为4类：
   - **公有报文**：显示核心字段（如电池厂商、剩余电量），点击可查看全字段详情
   - **私有报文**：识别报文ID并标注类型/发送周期
   - **校验**：显示和校验结果，计算值与接收值对比
   - **解析错误**：标注错误原因（如脉宽无效、校验失败）
3. 公有报文**详情面板**可查看：基础信息、额定参数、实时电压/电流/温度、故障码、工作状态等全字段。

## 关键解析说明
### 公有报文字段解析（协议表A.2）
| 字段         | 换算规则                     | 无效值       |
|--------------|------------------------------|--------------|
| 剩余电量     | 总线值 × 0.5%                | 0xFF         |
| 电压/额定电压| 总线值 × 0.1V                | 0xFFFF       |
| 额定容量     | 总线值 × 0.1AH               | 0xFFFF       |
| 工作电流     | (总线值 - 500) × 0.1A        | 0xFFFF       |
| 温度类       | 总线值 - 40℃                 | 0xFF         |

### 故障码标识说明（协议表A.4）
- **★**：整车端需30s内发出≥85dB(A)报警声音
- **▲**：整车端需做指示（无需声音）
插件解析结果直接携带标识，方便快速定位故障等级。

### 私有报文说明
私有报文ID/长度由主机厂自定义，插件已识别**0x3B(单串电压)**、**0x3C(电池唯一码)** 两类北京团标报文，其他私有报文需根据主机厂协议在`_parse_frame`方法中扩展解析逻辑。

## 常见问题排查
### 1. 无解析结果
- 检查GND是否共地，COM引脚接线是否正确
- 确认BMS与ECU/充电器已连接（BMS需检测到上拉电阻才发送报文）
- 采样率是否≥500Ksps，建议1Msps

### 2. 大量「脉宽无效」错误
- 硬件接线接触不良，重新插拔接线
- 信号存在干扰，增加屏蔽线
- 调整脉宽容差（修改代码中`PULSE_TOLERANCE`值，默认0.5ms）

### 3. 校验失败
- 报文传输过程中丢包/干扰，重新采集
- 确认插件版本与星恒BMS协议版本匹配（A1.3.3）

### 4. 部分字段显示「无效」
- 对应字段未被BMS采集/上报，属于正常现象
- 检查BMS硬件是否正常（如温度传感器、电压采集模块）

## 扩展开发
若需解析**主机厂自定义私有报文**，可修改`xingheng_bms_single_wire.py`：
1. 在`_parse_frame`方法中添加自定义报文ID的判断
2. 新增解析方法，按主机厂协议定义解析字段（参考`_parse_public_frame`）
3. 扩展`result_types`定义表格输出列

若需解析**单串电压/电池唯一码**的具体值，可在`_parse_frame`中对0x3B/0x3C报文添加解析逻辑，按协议表A.6/A.7换算。

## 注意事项
1. 本插件仅用于星恒BMS一线通协议解析，禁止用于其他品牌BMS
2. 解析结果仅作调试参考，不作为电池故障判定的唯一依据
3. 硬件操作时请遵守安全规范，避免短路、过压
4. 协议中加密算法/充电策略/整车互认策略为星恒保密内容，插件未包含相关解析
